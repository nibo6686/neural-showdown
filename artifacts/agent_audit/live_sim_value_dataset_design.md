# Live/Sim Value Dataset Design (Part A)

Audit date: 2026-06-18
Scope: seeded Gen 9 singles. Targeted fix for the value-model train/serve skew
(see `value_model_diagnostics.md`), not broad retraining.

## Findings (Part A questions)

- **Which feature builder for sim-core/live branch states?**
  `build_features_from_live_payload` (the live/sim serving path), **not** the
  replay-native reconstruction path. The prod model was trained on the latter and
  collapses on the former — that is the skew.
- **How should labels be perspective-correct?** Use
  `result_from_winner_side(winner_side, perspective=side)` ∈ {-1,0,+1} per viewed
  side, optionally discounted to the terminal via
  `discounted_terminal_return(result, turns_to_end, gamma)`. Record both p1 and p2
  perspectives per state for balance.
- **Where does the current checkpoint differ from the target distribution?** It
  was built with reconstructed private teams + replay-parsed public features; the
  serving path uses sim-core `ChoiceRequestView` (`side` as a Pokemon list) +
  sim-core spectator protocol + randbats beliefs. Same nominal 115D
  `live-private-belief-v2`, but a different empirical distribution → perspective
  signal lost (collapse to ~+1 both sides).
- **Reuse or new builder?** A **new** builder is cleaner. The existing builders are
  tied to replay/trace inputs; the fix needs states generated by sim-core games
  and featurized through the serving path. The new builder
  (`build_live_sim_value_dataset.py`) shares only the label helpers.

## Design adopted

- **Generator:** seeded Gen 9 Random Battle sim-core games, both players external,
  controllers alternating heuristic-mirror and heuristic-vs-random for state
  diversity (balanced + decisive games). Record per-turn `(public protocol, per-side
  request)` snapshots and the final winner.
- **Featurize** each actionable per-side state through `build_features_from_live_payload`
  (115D `live-private-belief-v2`) — exactly the serving path.
- **Labels:** `discounted_terminal_return(result_from_winner_side(winner, side),
  turns_to_end, gamma=0.97)`, bounded `[-1,1]`, perspective-correct, both sides.
- **No hidden info in features:** features are perspective-filtered (own request +
  public protocol + randbats beliefs); the dataset is live-style, not
  exact-seeded. (The branch *evaluator* may fork the exact opponent in research
  mode, but the value features here do not encode it.)
- **Bounded head:** `BoundedValueMLP` (tanh) trained with Huber on labels in
  `[-1,1]` (`train_live_sim_value.py`); a new checkpoint, no production overwrite.
- **Skip + count** non-actionable requests, missing winners, feature errors, and
  dimension mismatches.

Outputs: `data/value/gen9randombattle_live_sim_value_v1.npz`,
`artifacts/checkpoints/gen9randombattle_live_sim_value_v1.pt`, opt-in scorer
`NEURAL_BRANCH_SCORER=live_sim_value`.
