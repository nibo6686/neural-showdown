## `.gitignore` change

Appended one section:

```gitignore
# Generated agent-audit / validation run outputs.
# Keep hand-written .md reports and design docs; ignore per-battle JSON dumps.
artifacts/agent_audit/**/*.json
artifacts/validation/*.json
```

**Why:**
- Every `*.json` under `artifacts/agent_audit/` lives in a run subdirectory
  (`runs/`, `belief_particles/`, `two_ply_smoke/`, …) and is a regenerable per-battle dump
  or summary produced by the audit tooling. None are hand-authored.
- `artifacts/validation/sim_core_validation_results.json` is regenerated by
  `validate-sim-core`; the human-readable `.md` companion is kept.
- The pattern is deliberately scoped to `*.json` so the **hand-written `.md` reports and
  design docs** in those same directories stay tracked (they document the research
  checkpoint and the user wants them committed).

No existing source-, config-, test-, lockfile-, or doc-ignoring rules were added or
changed. `sim-core/package-lock.json` remains tracked (dependency pinning matters).
